How can Minecraft Commands / Datapack help you?

Function & Command_block

Description 실행 파일에 미리 작성된 명령을 실행합니다. 일반적으로 Command_block보다 성능이 좋고 청크를 소비하지 않습니다. 또한 글자 수의 제한이 없으며 주석(#)을 추가하여 명령마다 가독성을 향상시킬 수 있습니다. 실행 파일은 .mcfunction 형식이며 메모장 또는 Visual Studio Code와 같은 텍스트 편집기로 작성할 수 있습니다.

Description. 일반적으로 Function으로 이동하기 전에 많은 커맨더가 사용합니다. 하지만 Command_block만이 가지는 장점이 있습니다. 예를 들어, 여러 대수학을 적용하여 복잡한 문제를 해결하는 한 줄 명령과 1.19.4 업데이트로 가능해진 문자열 수정 명령이 있습니다. 특히 문자열 수정은 원래 해결할 수 없었던 일을 가능하게 합니다. (+ 참고로 Command_block의 출력에는 현재 시간이 표시됩니다.)

Command Optimization

Description. 여기서는 여러가지 최적화 방법들, 크게 대상 선택 인자, execute 에 한해서 그 작성 팁들까지 소개합니다.


참고 문헌 또는 자료.

Performance test [Reddit]

Command Efficiency Tier List [Youtube] Optimising [Reddit]

모든 커맨드의 시작점 마인크래프트에서 대상을 지정하는 데 사용합니다. 대상 선택의 종류는 총 5가지이며 다음과 같습니다.

@e <<< @a <<<< @r < @p <<<<<<<<<< @s 부등호는 성능의 지표를 간단하게 나타냅니다. 이에 따라 함수내에서 @s를 많이 사용해야 한다는 것을 알았습니다.


다음은 인자 내부의 자주 쓰는 6가지 세부 명령을 나열한 내용입니다.

nbt <<< predicate < advancements <<<<<< team << tag = scores

이 모든 측정은 @s[]를 기준으로 하며 측정 대상은 player와 marker입니다. 또한 tag는 한 단어든 매우 긴 단어든 성능의 차이가 미미하게 나타났고 NBT까지 감지하는 predicate는 세부 명령 nbt를 감지했을 때 보다 성능이 더 낮은 결과가 나왔습니다.

"execute가 있었기에 우리가 존재했다..."
마인크래프트에서 명령어 하면 빠질 수 없는 존재입니다. 명령어의 실행 주체, 위치, 시점, 차원을 설정하고 명령어가 실행될 조건과 명령어의 결과값을 저장하는 등 여러 기능을 수행할 수 있습니다.


하지만 단점도 존재합니다. 여러 기능을 수행할 수 있지만 비용(코스트 즉, 명령을 수행하였을 때 잡아먹는 성능)이 비싸 최적화가 꼭 필요한 명령이기 때문입니다. 그렇다면 어떻게 최적화를 해야 할까요?


1. 하나의 실행파일에 execute as를 중복 사용하지 말아라.

: 대부분의 커맨더들이 하고 있는 실수 중에 하나입니다. 파일 하나에 execute as ...가 많아질수록 성능이 떨어지게 됩니다. 이를 어느정도 막기 위해서는 실행 대상의 이름을 파일명으로 정하고(ex: player.mcfunction) 그 하위 명령에 @s인자를 사용하는 것입니다. 이렇게 한다면 성능도 상승할 뿐 아니라 가독성도 좋아질 것입니다.


2. 최대한 명령을 압축하여라.

: 만약 실행 대상의 test 점수가 1점이라면 test1의 점수를 1점으로 설정해 라는 명령을 실행한다고 해 봅시다. 아마도 이러한 명령은 다음과 같을 것입니다.

execute if score @s test matches 1 run scoreboard players set @s test1 1

하지만 우리는 이제 해당 명령을 압축할 수 있습니다. 바로

scoreboard players set @s[scores={test=1}] test1 1 이렇게요.

이것은 성능이 크게 바뀐다는 의미는 아니지만, 이전 명령보다는 조금이나마 효율적일 것입니다.
